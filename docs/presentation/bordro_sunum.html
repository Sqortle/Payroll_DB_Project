Örnek: Employee Tablosu

sql / 01_create_tables.sql · CREATE TABLE Employees
CREATE TABLE Employees (
    employee_id        NUMBER,
    fk_company_id      NUMBER NOT NULL,
    fk_department_id   NUMBER NOT NULL,
    fk_job_title_id    NUMBER NOT NULL,
    employee_code      VARCHAR2(30),                       
    national_id        VARCHAR2(11),                       
    first_name         VARCHAR2(50) NOT NULL,
    last_name          VARCHAR2(50) NOT NULL,
    birth_date         DATE,
    marital_status     VARCHAR2(20),
    children_count     NUMBER(2) DEFAULT 0,
    hire_date          DATE NOT NULL
);

ALTER TABLE Employees ADD CONSTRAINT pk_employees
    PRIMARY KEY (employee_id);
ALTER TABLE Employees ADD CONSTRAINT uq_emp_tckn,
    UNIQUE (fk_company_id, national_id);
①
Güvenlik
Aynı şirkette iki personelin TCKN'si veya kodu aynı olamaz. Bu kuralı UNIQUE ile veritabanına gömdük; programcı unutsa bile kayıt geçmez.
②
Her Satır Bir Şirkete Ait
Her kayıtta fk_company_id kolonu var; "bu personel hangi şirketin?" sorusunun cevabını taşır. Farklı şirketler aynı tabloda durur ama birbirine karışmaz.
③
Null Pointer Exception Önlemi
Çocuk sayısı yazılmazsa otomatik DEFAULT 0 olur. Hücre NULL kalmaz, maaş hesabında "tanımsız değer" hatası çıkmaz.

Otomatik Tetiklemeler

sql / 03_sequences_triggers.sql · trg_after_emp_insert
CREATE OR REPLACE TRIGGER trg_after_emp_insert
AFTER INSERT ON Employees
FOR EACH ROW
DECLARE
    v_month NUMBER := EXTRACT(MONTH FROM SYSDATE);
    v_year  NUMBER := EXTRACT(YEAR FROM SYSDATE);
BEGIN
    INSERT INTO Attendance_Records (
        attendance_id, fk_employee_id, fk_company_id,
        record_month, record_year,
        worked_days, overtime_hours
    ) VALUES (
        seq_attendance_id.NEXTVAL,
        :NEW.employee_id, :NEW.fk_company_id,
        v_month, v_year,
        0, 0
    );
END;
01 · Davranış

Otomatik Puantaj Kaydı

Personel EMPLOYEES'a eklendiği anda o ayki puantaj kaydı ATTENDANCE_RECORDS'da kendiliğinden oluşur. Mantık AFTER INSERT üzerinde, satır başına çalışır (FOR EACH ROW).

02 · Neden

İnsan Hatası Şemadan

"İK puantaj eklemeyi unuttu" hatasını şemanın kendisi ortadan kaldırır. Operasyonel hatanın kaynağı kod katmanında bırakılmıyor; veritabanı kendiliğinden eksiksiz çalışıyor.

03 · İkinci Trigger

Audit Otomasyonu

İkinci trigger trg_payroll_audit, bordro üzerindeki her DML'i PAYROLL_LOGS'a yazar. INSERTING / UPDATING / DELETING dahili değişkenleri ile tek trigger üç olayı ayırır.

Esnek Filtreler

sql / packages / pkg_payroll_reports.pkb · get_employee_report
-- Sorguya başlangıç
v_sql := 'SELECT ... FROM Employees e ' ||
         'WHERE e.fk_company_id = ' || p_company_id;

-- Departman seçildiyse WHERE bloğuna ekle
IF p_department_id IS NOT NULL THEN
    v_sql := v_sql || ' AND e.fk_department_id = '
                   || p_department_id;
END IF;

-- Unvan seçildiyse WHERE bloğuna ekle
IF p_job_title_id IS NOT NULL THEN
    v_sql := v_sql || ' AND e.fk_job_title_id = '
                   || p_job_title_id;
END IF;

OPEN v_cursor FOR v_sql;
RETURN v_cursor;
01 · Mekanizma

Çalışma Anında Sorgu Kurulumu

WHERE bloğunu çalışma zamanında, kullanıcının verdiği parametrelere bakarak inşa ediyoruz. Departman parametresi geldiyse sorguya AND e.fk_department_id ekleniyor; gelmediyse hiç katılmıyor. Tek bir fonksiyon, sınırsız filtre kombinasyonuna hizmet eder.

02 · Geri Dönüş

Cursor ile Akış

Fonksiyon, satırları doğrudan döndürmek yerine bir SYS_REFCURSOR referansı verir. Çağıran taraf cursor'ı açıp satırları teker teker okur — büyük sonuç kümelerinde bellek kullanımı minimumda kalır.

03 · İki Fonksiyon

Eksiksiz Raporlama

get_employee_report — şirket zorunlu, departman ve unvan opsiyonel; çalışan listesi alınır. get_monthly_payroll — şirket, ay, yıl zorunlu; personel opsiyonel; aylık bordro raporu alınır.

Mükerrer Kayıt Temizliği

sql / packages / pkg_payroll_maintenance.pkb · remove_all_duplicates
-- 1) Employees: aynı şirkette aynı TCKN'leri sil
DELETE FROM Employees
WHERE rowid NOT IN (
    SELECT MIN(rowid) FROM Employees
    GROUP BY fk_company_id, national_id
);

-- 2) Companies: NULL-safe varyant
DELETE FROM Companies
WHERE tax_number IS NOT NULL
  AND rowid NOT IN (
    SELECT MIN(rowid) FROM Companies
    WHERE tax_number IS NOT NULL
    GROUP BY tax_number
);

-- 3) Atomicity güvencesi
EXCEPTION WHEN OTHERS THEN
    ROLLBACK;
    RAISE_APPLICATION_ERROR(-20020,
        'Dedupe başarısız: ' || SQLERRM);
01 · Kapsam

15 Tablonun Tamamı

remove_all_duplicates tek bir prosedürdür ve şemanın 15 tablosunda da çalışır. Her tabloda business key gruplarına bakar; her gruptan yalnızca MIN(rowid) satırını tutar, kalan kopyaları siler.

02 · NULL-Safe

Companies İçin Özel Filtre

Companies'te tax_number NULL olabilir. Eğer NULL değerleri normal GROUP BY ile gruplarsak, tüm NULL satırlar tek bir grup sayılır — yanlış sonuç. Bu yüzden tax_number IS NOT NULL filtresini hem dış DELETE'te hem iç SELECT'te kullanırız.

03 · Atomicity

Hepsi veya Hiçbiri

Prosedürün sonunda tek EXCEPTION WHEN OTHERS bloğu var. 15 tablodan birinde hata olursa ROLLBACK çalışır ve önceki temizlikler de geri sarılır. Yarım bir temizlik durumu oluşmaz.

Birinci Aşama

Atomik Değerler

Her hücrede tek, bölünmez bir değer olur. Tekrarlayan sütun grupları (örn. çoklu telefon numaraları, aylık maaş kolonları) yasaktır — her satır bir birincil anahtarla tanımlıdır.

Bizim örneğimiz: Ocak_Maaşı, Şubat_Maaşı, Mart_Maaşı… şeklinde 12 sütun açmadık. Bunun yerine Payroll_Summary'de period_month + period_year ile her ay ayrı bir satır olarak tutulur.

Tam Bağımlılık

1NF sağlanmış olmalı + kısmi bağımlılık olmamalı. Yani PK olmayan her sütun, anahtarın tümüne bağlı olmalı, parçasına değil.

Bütünlük

İş kuralları PL/SQL kodu yerine doğrudan composite UNIQUE'lara gömüldü. İki trigger reflekssiz bırakmıyor; audit log her hareketi kaydediyor.

İşlem

Kanıt 04

Normal Form
